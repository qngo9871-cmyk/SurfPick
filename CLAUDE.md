# Surf Pick

iPhone-only app that picks the best surf spot near you right now from the 10 nearest breaks. Tap any spot to get directions in Google Maps (default) or Apple Maps. Includes a home-screen widget that automatically appears in CarPlay on iOS 26.

Sister app to **Surf Near Me** (watchOS, separate App Store listing). Both apps share the data layer via the `SurfShared` Swift Package at `/Users/user/SurfShared`.

## Identity

- **Name**: Surf Pick
- **Subtitle**: "Where to surf right now"
- **Bundle ID**: `com.quyenngo.surfpick`
- **Widget bundle ID**: `com.quyenngo.surfpick.widget`
- **Wedge vs Surfline**: simple, no subscription, decision tool not browser
- **Wedge vs Dawn Patrol**: ranked-list-of-nearest-breaks, not "your tracked spots"

- iPhone-only iOS app (Swift / SwiftUI), iOS 17+

## Initial Xcode setup (one-time, manual)

The Xcode project is created in Xcode and not generated by CLC because `.xcodeproj` files are too brittle to hand-author. Once it exists, CLC handles every code change.

1. **Create the iOS app project** in Xcode:
   - File → New → Project → iOS App
   - Product Name: `SurfPick`
   - Bundle Identifier: `com.quyenngo.surfpick`
   - Interface: SwiftUI
   - Language: Swift
   - Save to `/Users/user/SurfPick/`
   - **Delete** the auto-generated `SurfPickApp.swift` and `ContentView.swift` from the project (we have our own).

2. **Add the SurfShared local package**:
   - File → Add Package Dependencies → Add Local
   - Choose `/Users/user/SurfShared`
   - Add to target: `SurfPick`

3. **Drag the source files in** from `/Users/user/SurfPick/SourceFiles/` (excluding the `Widget/` subfolder) into the Xcode project navigator under the `SurfPick` group. Make sure "Copy items if needed" is unchecked and they're added to the `SurfPick` target.

4. **Add the widget extension target**:
   - File → New → Target → Widget Extension
   - Product Name: `SurfPickWidget`
   - Bundle Identifier: `com.quyenngo.surfpick.widget`
   - Include Configuration App Intent: **NO**
   - Embed in: SurfPick app
   - Activate scheme: Yes
   - **Delete** the auto-generated widget files from the new target (we have our own).
   - Drag in `Widget/SurfPickWidget.swift` and `Widget/SurfPickWidgetBundle.swift` from `/Users/user/SurfPick/SourceFiles/Widget/` — add to **SurfPickWidget target only**.
   - Add SurfShared package dependency to the widget target as well (Target → Frameworks, Libraries → +).

5. **Configure Info.plist for the SurfPick target**:
   - `NSLocationWhenInUseUsageDescription` → "Surf Pick uses your location to find the best surf spots near you right now."
   - `LSApplicationQueriesSchemes` → array containing `comgooglemaps`
   - `UIRequiredDeviceCapabilities` → array containing `arm64` (Apple's Guideline 2.3 requirement)

6. **App icon**: drop a 1024×1024 icon into `Assets.xcassets/AppIcon.appiconset` (Xcode 14+ auto-generates the rest from a single icon).

7. **Run on device**: select the SurfPick scheme, pick your iPhone as the destination, hit ⌘R.

## Design philosophy

- **Decisiveness, not data.** Surfline gives you data and lets you decide. Surf Pick decides. The #1 break is visually dominant on the main screen.
- **Surfers want to go, not graph.** No long charts on the main screen. The detail screen has the data for those who want it.
- **One refresh button (top-right) plus pull-to-refresh.** Conditions change quickly so refresh is one-tap.
- **Traffic-light rating before any numbers.** Green/Amber/Red is the headline. Wave height in metres is supporting detail.
- **Get Directions is the most prominent action.** Defaults to Google Maps (most surfers' default). Settings lets users switch to Apple Maps. Detail screen has both buttons.
- **No subscription in v1.** Apple's Guideline 4.3 (Spam) defence vs Surf Near Me requires a clearly different product, so we lean into iPhone-specific UX (ranked list + directions + widget) and pricing simplicity.

## Differentiation from Surf Near Me (watchOS)

To pre-empt Apple Guideline 4.3 (Spam) review concerns:

| Aspect | Surf Near Me (watchOS) | Surf Pick (iPhone) |
|---|---|---|
| Platform | Apple Watch only | iPhone (+ widget + CarPlay widget) |
| UX | Glance, 5-second use, Digital Crown | Decision tool, ranked list, drive directions |
| Spot count | 1 visible at a time | 10 visible, ranked |
| Sort order | Geographic (south-to-north for crown scrolling) | By quality (best surf first) |
| Directions | Not applicable | Google Maps default + Apple Maps |
| Conditions data | Live for one spot | Live for 10 spots in parallel |
| Widget | watchOS complication | iPhone home-screen + auto CarPlay |
| Wedge | Glanceable | Decisive |

App Store listings for both apps must lean hard into platform-specific screenshots and descriptions to make this differentiation visible at a glance.

## Pricing model (decided 2026-04-25)

Free + one-time Pro unlock ($4.99). NO subscription — that contradicts the anti-Surfline wedge.

**Free tier:**
- Top 3 nearest spots, ranked by quality (the wedge is visible)
- Get Directions on all 3
- Traffic-light rating
- Refresh button

**Pro tier ($4.99 one-time, StoreKit 2):**
- Top 10 nearest spots
- Home-screen widget (auto-becomes CarPlay widget on iOS 26)
- Custom radius (e.g. willing to drive 90 min for a green-rated break)
- Saved trip presets (home/work/travel locations)
- Possibly: "Notify when this break goes green" push notifications (v1.1)

Free version delivers the core promise (the ranked decision is visible). Pro adds depth, convenience, and quantity. Reviews-positive freemium structure.
